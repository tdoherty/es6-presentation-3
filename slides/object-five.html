let obj1 = {
      foo() {
        return 'bar';
      }
    };

    let obj2 = {
      bar: '2'
    }

    Object.setPrototypeOf(obj2, obj1);
    obj2.foo()   // 'bar'

    * Object.create() is is still preferred
Assigning a new prototype to an existing object precludes compiler optimizations.
Setting the prototype at creation time will always be faster
Alternative to controversial (but now standardized) __proto__ getter/setter
